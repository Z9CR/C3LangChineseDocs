C3有四种注释:

1. `\\`样式的单行注释
2. C样式的多行注释`\* *\`
3. 像 `<* ... *>` 一样的文档注释, 内部的注释将会被注释成文档
4. shebang注释`#!`, 其只在文件开头可用

## 文档注释

文档注释以`<*`开始, `*>`结束
任何以`@`开头的行将会被视为说明的开始

如下:

```c3
<*
 这是一个文档.
 @param num_foo : `函数参数`
 @require num_foo > 4
 @deprecated
 @mycustom "2"
*>
fn void bar(int num_foo)
{
    io::printfn("%d", num_foo);
}
```

### 文档契约会被解析

以下内容已被提取:
- 函数描述: *"这里是一些文档."*
- `num_foo` 参数的描述: *"foo 的数量"*.
- 针对编译器的契约注解: `@require num_foo > 4`, 它告诉编译器和函数的使用者, 前置条件是 `num_foo` 必须大于 4.
- 第二个契约注解, 附带描述: *"防止过多的 foo"*.
- 一个函数属性, 标记为 `@deprecated`, 会显示警告.
- 一个自定义函数属性 `@mycustom`. 编译器可以自由地忽略自定义属性, 它们可用于选择性地发出警告, 否则将被忽略.

### 可用的注解

| 名称           |                                              格式 |
|----------------|--------------------------------------------------:|
| `@param      ` |          `@param [<ref>] <param> [ : <description>]` |
| `@return     ` |                              `@return <description>` |
| `@return?    ` | `@return? [<func>!], [<fault1>, <fault2>, ..., [: <description>]]` |
| `@require    ` |    `@require <expr1>, <expr2>, ..., [: <description>]` |
| `@ensure     ` |    `@ensure <expre1>, <expr2>, ..., [: <description>]` |
| `@deprecated ` |                        `@deprecated [<description>]` |
| `@pure       ` |                                              `@pure` |

### 错误继承

可以使用语法 `@return? some_func!` 来引用另一个函数或宏的错误. 这将包含 `some_func` 返回的所有错误. 这可以与其他错误组合使用.

```c3
<*
 @return? check_triangle!, io::EOF
*>
fn TriangleKind? get_triangle_kind(Triangle* triangle)
{
    check_triangle(triangle)!;
    // ...
}
```

关于 `@require`, `@ensure`, `@const`, `@pure` 的信息, 请参阅契约
\*`[<ref>]` 是一个可选的可变性描述, 例如 `[&in]`
\*`[<description>]` 表示描述是可选的.
