C3像C一样也提供了很多基础数据类型如: 整数, 浮点, 数组 和 指针.
同时拓展了 向量 和 切片, 同时也提供 `any` 与 `typeid` 给高端用户.

## 整数

C3 具有有符号和无符号的整数类型。内建有符号整数类型包括 `ichar`、`short`、`int`、`long`、`int128`、`iptr` 和 `isz`。`ichar` 到 `int128` 都具有明确的 2 的幂次位宽，而 `iptr` 的位数与 `void*` 相同，`isz` 的位数与两个指针之间的最大差值相同。对于每个有符号整数类型，都有对应的无符号整数类型：`char`、`ushort`、`uint`、`ulong`、`uint128`、`uptr` 和 `usz`。

| 类型    | 有符号? |  最小值| 最大值    |大小/bit|
|---------|---------|--------|-----------|--------|
| ichar   | yes     | -128   | 127       | 8      |
| short   | yes     | -32768 | 32767     | 16     |
| int     | yes     | -2^31  | 2^31 - 1  | 32     |
| long    | yes     | -2^63  | 2^63 - 1  | 64     |
| int128  | yes     | -2^127 | 2^127 - 1 | 128    |
| iptr    | yes     | varies | varies    | varies |
| isz     | yes     | varies | varies    | varies |
| char    | no      | 0      | 255       | 8      |
| ushort  | no      | 0      | 65535     | 16     |
| uint    | no      | 0      | 2^32 - 1  | 32     |
| ulong   | no      | 0      | 2^64 - 1  | 64     |
| uint128 | no      | 0      | 2^128 - 1 | 128    |
| uptr    | no      | 0      | varies    | varies |
| usz     | no      | 0      | varies    | varies |

x64机器上 `iptr`/`uptr` 和 `isz`/`usz` 是恒定64bit的, 跟 `long`/`ulong` 一样.
而x32机器上和 `int`/`uint` 大小相同.

### 常量

一般我们写的字面量都是10进制的, 如 `234`, 但是我们一样可以通过前缀 `0x`(不区分大小写) 声明一个16进制字面量, 如 `int a = 0x42edaa02;`.也可以通过 `0o` 和 `0b` 声明8进制或2进制数 

数字也可以被下划线所分割, 如 `1_000_000`.

```c3
a = -2_000;
b = 0o770;
c = 0x7f7f7f;
```

对于十进制数，其值被认为是 `signed int` 类型，除非该数字无法放入 `int` 中，此时它将被认为是能够容纳该数字的最小有符号类型（`long` 或 `int128`).

### 常量前缀

如果你想要确保一个常量是某种特定类型，你可以添加一个显式转换，例如：`(ushort)345`，或者使用整数后缀 `345u16`

可用的整数后缀如下

| suffix |    type |
|--------|--------|
| i8     |   ichar |
| i16    |   short |
| i32    |     int |
| i64    |    long |
| i128   |  int128 |
| u8     |    char |
| u16    |  ushort |
| u32    |    uint |
| u      |    uint |
| u64    |   ulong |
| u128   | uint128 |

## 布尔值

`bool` 要么是 `true` or `false`. 尽管布尔值在数据上只占一个比特位, 但需要注意的是, 它实际存储在一个字节中

```c
bool b = true;
bool f = false;
```

### 字符字面量

字符字面量是用 `''` 括起来的值. 对于单个字符, 其值被解释为该字符的 ASCII 值.

也可以使用 2, 4 或 8 个字符宽的字符字面量, 它们分别被解释为 ushort, uint 和 ulong, 并在内存中从左到右排列. 这意味着实际值取决于目标平台的[字节序](https://en.wikipedia.org/wiki/Endianness).
 * 2 字符字面量, 例如 'C3', 将转换为 ushort.
 * 4 字符字面量, 例如 'TEST', 将转换为 uint.
 * 8 字符字面量, 例如 'FOOBAR11', 将转换为 ulong.
 * 4 字符字面量对应于 [FourCC](https://en.wikipedia.org/wiki/FourCC) 码的布局. 它还能正确地在内存中排列 Unicode 字符. 例如 Char32 smiley = '\u1F603'.

## 浮点类型

C3和其他语言一样有两个浮点类型: `float` 和 `double`. `float` 为单精度32位浮点, 
`double` 为双精度64位浮点

### 浮点字面量

浮点字面量*至少*为双精度.
正如整数常量一样, 可以使用 `_` 来提高可读性, 但它不能出现在小数点或指数符号的前后.

C3支持10/16进制浮点.
10进制的指数符号是`e` (或 `E`),
16进制是`p` (或者`P`)

例子:
`-2.22e-21` `-0x21.93p-10`

虽然浮点字面量默认双精度, 但是你可以添加后缀使得其变成单精度:

|后缀|类型|
|-|-|
|`f32`*或者*`f`|`float`|
|`f64`|`double`|

## 数组

[数组](/language-common/arrays/) 按照格式 `Type[size]` 书写, 数组是同种类型的变量的集合. `int[4]` 是4个 `int` 值组成的.

在初始化时, 有时使用通配符 `Type[*]` 声明会很方便, 它会根据元素个数推断长度:

```c3
int[3] abc = { 1, 2, 3 }; // Explicit int[3]
int[*] bcd = { 1, 2, 3 }; // Implicit int[3]
```

## 切片

[切片](/language-common/arrays/#slice) 按照格式 `Type[]`. 与数组不同, 切片本身并不存储值, 而是提供对某个底层数组或向量的视图.

切片有两个属性: `.ptr` 用于获取它所指向的数组, 以及 `.len` 表示切片的长度, 即可以索引访问的元素个数.

通常我们可以通过取数组的地址来获得一个切片:

```c3
int[3] abc = { 1, 2, 3 };
int[] slice = &abc;       // 一个长度=3指向abc的切片
```

因为在安全模式下对切片的索引会进行范围检查, 所以将指针和长度分开提供的切片要安全得多. 切片的内部表示是一个双元素结构体:
```c3
struct SliceRaw
{
	void* ptr;
	usz len;
}
```
你能在 `std::core::runtime` 找到切片的定义.

## 向量

[向量](/language-common/vectors/) 和数组相似, 按照格式
`Type[<size>]`, 但限制向量只能由整数, 浮点数和布尔值组成. 与数组类似, 可以使用通配符来推断向量的大小.

```c3
int[<*>] a = { 1, 2 };
```

向量基于硬件 SIMD 向量, 并支持许多并行作用于所有元素的不同操作, 包括算术运算:
```c3
int[<2>] b = { 3, 8 };
int[<2>] c = { 7, 2 };
int[<2>] d = b * c;    // d 为 { 21, 16 }
```

向量一样可以跟数组用一个初始化方式(`{ args... }`), 
空转参数也行

```c3
int[<3>] v = { .xy = 3, .z = 100 }; // 等价于 { 3, 3, 100 }
```

## Base64 和十六进制数据字面量
Base64 字面量是以 `b64` 为前缀的字符串, 其中包含 [Base64 编码](https://en.wikipedia.org/wiki/Base64) 的数据, 并在编译时转换为字符数组:

```c3
// 下面的数组包含字符 "Hello World!"
char[*] hello_world_base64 = b64"SGVsbG8gV29ybGQh";
```
相应的十六进制数据字面量转换的是十六进制字符串, 而不是 Base64:
```c3
// 下面的数组包含字符 "Hello World!"
char[*] hello_world_hex = x"4865 6c6c 6f20 776f 726c 6421";

```
## 指针

指针写作 `Type*`. 指针是一个内存地址, 其中存储了底层地址中的一个或多个可能的元素. 指针可以嵌套: `Foo*` 是指向 `Foo` 的指针, 而 `Foo**` 是指向 `Foo` 的指针的指针.

指针类型有一个特殊的字面量 `null`, 表示一个无效的空指针.

### `void*`

`void*` 类型是一种特殊的指针, 它可以隐式转换为任何其他指针. 它不是"指向 void 的指针", 而是一个通配符指针, 可以匹配任何其他指针.

## 打印值

可以使用 `io::print`, `io::printn`, `io::printf` 和 `io::printfn` 来打印值. 这需要导入模块 `std::io`.

注:
打印函数的 `n` 变体会在打印后添加一个换行符, 这也是我们在示例中经常使用的形式, 但 `print` 和 `printf` 的工作方式相同.

```c3
import std::io; // 获取 io 函数.

fn void main()
{
    int a = 1234;
    ulong b = 0xFFAABBCCDDEEFF;
    double d = 13.03e-04;
    char[*] hex = x"4865 6c6c 6f20 776f 726c 6421";
    io::printn(a);
    io::printn(b);
    io::printn(d);
    io::printn(hex);
}
```

运行该程序将得到:

```
1234
71963842633920255
0.001303
[72, 101, 108, 108, 111, 32, 119, 111, 114, 108, 100, 33]
```

为了获得更精细的控制, 我们可以使用 `printf` 和 `printfn` 来格式化输出:

```c3
import std::io;
fn void main()
{
    int a = 1234;
    ulong b = 0xFFAABBCCDDEEFF;
    double d = 13.03e-04;
    char[*] hex = x"4865 6c6c 6f20 776f 726c 6421";
    io::printfn("a was: %d", a);
    io::printfn("b in hex was: %x", b);
    io::printfn("d in scientific notation was: %e", d);
    io::printfn("Bytes as string: %s", (String)&hex);
}
```

我们可以应用[标准 printf 格式化规则](https://en.cppreference.com/w/c/io/fprintf), 但与 C/C++ 不同的是, 使用 `%d` 时无需指明类型——它可以打印从 `int` 到 `int128` 的有符号和无符号数, 实际上 `io::printf` 不支持 `%u`, `%lld` 等. 此外, `%s` 不仅适用于字符串, 还适用于任何类型:

```c3
import std::io;

enum Foo
{
    ABC,
    BCD,
    EFG,
}
fn void main()
{
    int a = 1234;
    uint128 b = 0xFFEEDDCC_BBAA9988_77665544_33221100;
    Foo foo = BCD;
    io::printfn("a: %s, b: %d, foo: %s", a, b, foo);
}
```

这将打印:

```
a: 1234, b: 340193404210632335760508365704335069440, foo: BCD
```
