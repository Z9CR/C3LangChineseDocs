# Install C3 Compiler Binary
---

## 二进制包:
* [Windows](# Windows)
* [Mac Arm64](# Mac)
* [Ubuntu](# Ubuntu)
* [Debian](# Debian)
* [Arch](# Arch)

## Windows
1. 下载[C3编译器](https://github.com/c3lang/c3c/releases/latest/download/c3-windows.zip)
2. 并将其解压至一个文件夹
3. (可选)将此文件夹加入环境变量

---

## Mac

1. 确保你有Xcode以及其命令行工具.
2. 下载[c3c](https://github.com/c3lang/c3c/releases/latest/download/c3-macos.zip)
3. 解压可执行文件与依赖库
4. 运行 ./c3c.

---

## Ubuntu

1. 下载[c3c](https://github.com/c3lang/c3c/releases/latest/download/c3-linux.tar.gz)或者[debug版](https://github.com/c3lang/c3c/releases/latest/download/c3-linux-debug.tar.gz)
2. 解压
3. 执行./c3c.

---

## Debian

1. 下载[c3c](https://github.com/c3lang/c3c/releases/latest/download/c3-linux.tar.gz)或者[debug版](https://github.com/c3lang/c3c/releases/latest/download/c3-linux-debug.tar.gz)
2. 解压
3. 执行./c3c.

## Arch

c3c在AUR上有软件包: c3c-git.

请使用aur助手下载:

```bash
paru -S c3c-git
# 或 yay -S c3c-git
# 或 aura -A c3c-git
```
手动克隆:

```bash
git clone https://aur.archlinux.org/c3c-git.git
cd c3c-git
makepkg -si
```

## 遇到问题？

如果你遇到类似 `No module named 'std::io' could be found` 的错误，可能需要设置 `C3C_LIB` 环境变量，指向标准库所在位置：

bash/zsh:
```bash
export C3C_LIB=/path/to/c3c/lib
```

fish:
```fish
set -gx C3C_LIB /path/to/c3c/lib
```

Windows(PowerShell):
```powershell
$env:C3C_LIB = "C:\path\to\c3c\lib"
```

## “cc: not found”

Linux&Macos上, C3可用的C编译器以链接C库. 虽然C3自带了一个链接器, 但若没有可用的C编译器, 你的系统很可能缺乏一个完整的编译环境

Linux用户通常应根据所用发行版的文档安装GCC或Clang, 以下列表是官方测试过的发行版以及编译链接C3程序所需的最小软件包集合:

|发行版|需要的包|命令|
|:-|:-|:-|
|Ubuntu/Debian|`gcc, libc6-dev`|`sudo apt-get install gcc libc6-dev`|
|Fedora/Rocky|`gcc`|`sudo dnf install gcc`|
|Arch Linux|`gcc`|`sudo pacman -S gcc`|
|OpenSUSE|`gcc, glibc-devel`|`sudo zypper install gcc glibc-devel`|
|Alpine Linux|`gcc, musl-dev`|`sudo apk add gcc musl-dev`|
|Void Linux|`gcc`|`sudo xbps-install -S gcc`|
